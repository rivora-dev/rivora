# RFC Template

Copy this file to `docs/rfc/RFC-XXXX-<slug>.md` to propose a new design.

```markdown
# RFC-XXXX

- Title:
- Author:
- Date:
- Status: Draft | Review | Approved | Rejected | Implemented

---

# Summary

Describe the proposal in one or two paragraphs.

---

# Problem

What problem are we solving? Who experiences it? Why does it matter?

---

# Goals

- ...

---

# Non-Goals

Explicitly state what this RFC is NOT trying to solve.

---

# Proposed Solution

Describe the solution. Include diagrams if helpful. Explain how it fits
into the existing architecture (see docs/04-Architecture.md).

---

# User Experience

How will engineers interact with this feature? CLI? Slack? Receipts?
Examples encouraged.

---

# Technical Design

Architecture, data flow, storage, interfaces, dependencies, testing
strategy, failure handling.

---

# Safety Considerations

Could this affect production systems? Could recommendations be misleading?
How do engineers maintain control? How is uncertainty communicated?
(See ADR-0002, ADR-0003, ADR-0009.)

---

# Alternatives Considered

Alternatives and why they were rejected.

---

# Tradeoffs

Benefits, costs, risks, complexity, future maintenance.

---

# Migration

Does this require migration? Breaking changes? Configuration updates?

---

# Future Work

Ideas intentionally left out of this proposal.

---

# Open Questions

Anything unresolved before implementation.

---

# Decision

Approved by:
Date:
Notes:
```
