# GitHub Labels

> Recommended labels for issues and pull requests. This is guidance only; no
> GitHub API automation is required. Create these manually in the repository
> label settings if desired.

---

## Type labels

| Label | Color suggestion | Purpose |
|---|---|---|
| `bug` | `#d73a4a` | Something is not working as expected |
| `feedback` | `#fbca04` | General product feedback |
| `design-partner` | `#5319e7` | Structured design partner report |
| `connector-request` | `#1d76db` | Request for a new evidence connector |
| `slack` | `#4a154b` | Slack-related issue or question |
| `docs` | `#0075ca` | Documentation improvement |
| `security` | `#b60205` | Security-related (use private reporting for vulns) |
| `good-first-issue` | `#7057ff` | Good for newcomers |
| `help-wanted` | `#008672` | Extra attention is needed |

## Concept labels
